# 执行限制实现方案

本文档详细介绍了 SalmonLoop 中执行限制的实现机制，用于防止程序失控并管理资源消耗。

## 核心机制

执行限制在多个层面上强制执行，以确保系统的稳定性和防御性运行。

### 基于时间的限制

- **总执行超时**：单次 `SalmonLoop` 运行允许的最大时间。
- **阶段超时**：各个阶段（规划、补丁、验证）都有各自的可配置超时时间。
- **LLM 调用超时**：防止 API 调用挂起导致的异常。

### 基于迭代的限制

- **最大重试次数**：在尝试修复问题达到指定次数后，循环将终止。
- **上下文收缩限制**：限制减小上下文窗口的次数。

### 基于资源的限制

- **内存监控**：循环监控自身的内存使用情况，如果超过阈值则终止。
- **Diff 大小限制**：在应用补丁前进行安全性检查，防止产生大规模非预期更改。

## 配置项

限制通过 `src/core/types.ts` 中的 `ExecutionLimits` 接口进行配置。

```typescript
export interface ExecutionLimits {
  maxRetries: number;
  totalTimeoutMs: number;
  phaseTimeoutMs: number;
  maxDiffSizeBytes: number;
  memoryLimitBytes: number;
}
```

## 防御性检查

`Monitor` 类负责在执行过程中跟踪这些限制。如果违反任何限制：

1. 当前操作将立即停止。
2. 系统尝试进行安全回滚（如果正在使用工作树或已启用检查点）。
3. 通过 `SalmonLoopResult` 返回详细的错误信息。

## 最近的改进 (阶段 10)

在阶段 10 中，我们通过以下方式增强了这些限制：
- **脏工作区保护**：确保现有更改被暂存或在工作树中隔离。
- **失败率监控**：跟踪检查点创建和清理失败的情况，以识别基础设施问题。
