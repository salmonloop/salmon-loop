## 2024-05-18 - Replacing Hardcoded Colors with Theme Tokens
**Learning:** In Ink-based terminal interfaces (like the sidebar components in salmon-loop), directly using string colors like "white", "gray", or "cyan" causes visual inconsistencies and tech debt. By replacing these strings with centralized semantic variables (like `COLORS.text.primary`, `COLORS.text.muted` from `theme.ts`), we ensure uniform contrast and simplify global theme updates, leading to a more consistent visual hierarchy across the CLI.
**Action:** When updating or creating new UI components using Ink's `<Text>` or `<Box>`, avoid hardcoded string colors. Always import the central `COLORS` object from the theme and use semantic values to ensure the interface remains cohesive and maintains standard contrast ratios.
