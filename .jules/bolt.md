## 2024-10-24 - Controlled Concurrency for File Operations
**Learning:** Using `Promise.all` directly to concurrently map arbitrary lists of files for metadata or states can lead to standard Node.js `EMFILE` errors under heavy load, specifically when attempting to process hundreds or thousands of files simultaneously (like when loading historical sessions or retrieving workspace maps). This pattern crashes or delays startup significantly due to resource exhaustion in the underlying OS.
**Action:** When performing `Promise.all` inside `map` for File System operations across a large number of dynamic items, strictly wrap the task within `processInBatches` (found in `src/core/utils/batch.ts`) and chunk the array (e.g. batch size of 10) to limit concurrency and improve runtime stability without negatively impacting overall speed.
