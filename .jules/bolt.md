## 2025-05-25 - ContextCacheStore Batch Eviction
**Learning:** In ContextService, sequentially awaiting cacheStore.delete(key) within loops for cache evictions (both TTL and LRU) leads to significant I/O overhead and performance bottlenecks. Each delete() operation in PersistentContextCacheStore triggers an atomic flushToDisk() which writes the entire cache state to disk. Running this in a for...of loop creates extreme I/O contention, causing the app to lock up and significantly degrading performance.
**Action:** When performing multiple async evictions on PersistentContextCacheStore, always batch operations. We improved the evictExpiredEntries and evictLruIfNeeded methods in src/core/context/service.ts to first compute the array of victims, and then delete them using batched Promise.all with a chunk size of 10. This limits the number of concurrent I/O operations, reducing evictExpiredEntries from ~1406ms to ~23ms and evictLruIfNeeded from ~1100ms to ~10ms for a cache with ~5000 items.
