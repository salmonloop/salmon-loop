## 2026-05-19 - [ContextService] Optimize cache eviction loops and batch deletions
**Learning:** In cache managers like `ContextService`, repeated asynchronous calls in loops, especially to fetch entries iteratively (`this.cacheStore.entries()`) or delete entries without chunking, lead to severe I/O contention. In `PersistentContextCacheStore`, the `.delete()` method triggers `flushToDisk()` which writes the entire cache to disk atomically. Triggering this repeatedly without batching leads to `EMFILE` errors (too many open files) and an O(M*N) time complexity for eviction loops.
**Action:** When handling mass cache expirations or evictions, fetch entries once using `Array.from()` to safely consume the iterable and avoid mutating internal references during operations like `.sort()`. Process expirations and deletions using batching (e.g., chunked `Promise.all` with a chunk size of 10) instead of unbounded concurrency to prevent EMFILE errors. For single evictions, use a linear O(N) scan; reserve O(N log N) sorting for bulk evictions. Instantiate Promises directly within `Promise.all()` for the active chunk to prevent upfront concurrent execution.
