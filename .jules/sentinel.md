## 2025-02-14 - Fix Command Injection Risk in Tool Authorization
**Vulnerability:** Found a command injection risk in `src/cli/authorization/non-interactive.ts` where `execa` is called with `shell: true` and the command (`cmd`) is taken directly from the configuration without any sanitation or parsing. This allows for arbitrary command execution.
**Learning:** The project's architecture memories specify: "To mitigate command injection risks in tool authorization, the project avoids using `shell: true` in `execa` calls in `src/cli/authorization/non-interactive.ts`. The configuration supports an optional `args: string[]` field (in `ToolAuthorizationConfig`) for explicit arguments, while a fallback `splitCommand` utility in `src/core/utils/command-split.ts` safely parses command strings when `args` are not provided." `shell: true` combined with an unsanitized command string opens the door to arbitrary command execution if the config is manipulated or unsafe.
**Prevention:** Avoid `shell: true` with user-provided commands. Execute the program and its arguments explicitly as an array. Use `splitCommand` to safely split string commands if explicit arguments are not provided.
