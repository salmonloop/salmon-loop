## 2025-03-01 - Fix Timing Attack Vulnerability in Bearer Token Validation
**Vulnerability:** Timing attack vulnerability during Bearer token validation in the Express auth middleware (`src/cli/commands/serve.ts`). The original code used `Array.prototype.includes` which delegates to a fast-failing `===` comparison string equality check.
**Learning:** Checking secure tokens utilizing string-based exact comparison operators (`===`, `!==`, `includes`) leaves the validation process open to side-channel timing attacks, as attackers can deduce the valid token one character at a time based on how quickly the comparison fails. This existed because simple array inclusion was chosen over explicit buffer comparison when adding token authentication to the API middleware.
**Prevention:** Always use `crypto.timingSafeEqual` after strictly matching the lengths of compared string values when evaluating sensitive secrets or tokens like passwords, Bearer tokens, API keys, or JWT signatures.
