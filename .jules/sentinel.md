## 2025-05-18 - Prevent Command Injection via `shell: true`
**Vulnerability:** A command injection risk existed in `src/cli/authorization/non-interactive.ts` where `execa` was being called with `shell: true` using an unvalidated command string from `params.config.nonInteractive?.command?.cmd`. This allows execution of arbitrary shell commands.
**Learning:** Using `shell: true` in standard child process or `execa` calls inherently passes the input string directly to the platform's default shell (like `sh` or `cmd.exe`). If any part of that command string can be influenced by untrusted input (e.g. from a configuration file that might be tampered with or user input), it creates a severe command injection vulnerability.
**Prevention:** To avoid this next time, never use `shell: true` in `execa` or child process methods when executing user-configurable commands. Instead, use the `getPlatformShellInvocation` utility from `src/core/utils/platform-shell.ts` to safely construct the shell executable and its arguments as an array, or always pass a strict arguments array directly to `execa` without `shell: true`.
