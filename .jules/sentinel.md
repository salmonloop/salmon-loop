## 2026-05-19 - Timing Attack Vulnerability in Token Validation\n**Vulnerability:** A timing attack vulnerability was identified in the `authTokens.includes(token)` standard string comparison for authorization tokens.\n**Learning:** Standard string comparisons or array prototype `includes` methods fail early, which can theoretically allow attackers to guess valid tokens character-by-character by measuring the server's response time.\n**Prevention:** When comparing secrets (like tokens or passwords), use a constant-time comparison such as `crypto.timingSafeEqual()` and ensure you compare buffers of the same length to avoid throwing TypeErrors.
