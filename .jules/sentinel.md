## 2025-02-24 - [CRITICAL] Fix arbitrary command execution vulnerability in MicroTaskRunner
**Vulnerability:** The legacy `MicroTaskRunner.ts` was taking command strings dynamically parsed from Markdown skills and directly executing them using `execa` with `getPlatformShellInvocation`. Since these strings are effectively user-provided or dynamically compiled, this allowed arbitrary shell command injection if the environment guard (NODE_ENV !== 'test') was bypassed or removed.
**Learning:** Even in deprecated, test-restricted legacy runners, executing un-sanitized commands through `execa(..., shell: true)` patterns is a major security risk. Testing mocks must never invoke physical execution tools.
**Prevention:** Remove real command execution wrappers entirely from legacy test harnesses. Use explicit mock returns (e.g., matching on specific strings like `'echo "ping"'`) instead of executing commands in the shell. Always review test harnesses to ensure they do not create physical side effects or pose supply chain injection risks.
